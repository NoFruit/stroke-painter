# 残差图区域分割：聚类特征选"标量强度"还是"RGB 三维"

**研究问题**：在残差/误差图 D = |T − C|（三通道 dR/dG/dB，∈[0,1]）上做 SLIC 超像素分割，
聚类特征该用 **方案 A（标量 v = mean(dR,dG,dB)，特征空间 (x,y,v)，3 维）** 还是
**方案 B（RGB 三维，特征空间 (x,y,dR,dG,dB)，5 维）**？

下游：选"差异最大区域"→ 在该区域算图像矩（质心/主轴/轴长/颜色）→ 给一笔画参数的初始建议，
之后由优化器（ot-brush-optimize）精修。

---

## 定夺

**推荐方案 A（标量强度）。**

**一句话理由**：残差图的语义是"修正量"（标量 quantity），不是"外观"（appearance）——
聚类应按"缺多少 + 在哪"分组，"缺什么色"的方向是输出量（笔颜色），应在区域确定后用区域内均值算，
而非作为分组键；且 SLIC 灰度模式天然 2 空间 +1 值平衡，避免 5 维下颜色 3 票压过位置 2 票的失衡与跨迭代权重漂移。

**逃生舱 → B**：仅当实测发现"被选中的高误差区域频繁跨越目标颜色边界、单方向笔颜色被均值成脏色、
且下游优化器也救不回来"时，才切到 B 并按下文公式重平衡权重。

---

## 文献要点（查证所得）

### 1. 经典 SBR：放置/误差信号一致用标量幅值

| 方法 | 放置信号 | 类型 | 笔颜色来源 |
|---|---|---|---|
| Haeberli (1990) | 全局 L2 标量损失（接受/拒绝扰动） | 优化 | 目标图 |
| Litwinowicz (1997) | 规则网格 + 边缘（标量） | 贪心 | 目标像素色 |
| Hertzmann (1998) | 边缘强度（标量）→ 密度图 | 贪心 | 目标图 |
| Image Painter (2026) | CCL on 阈值化幅值 + PCA 每分量 | 规则 | 区域均值 |

来源：Stroke-Based Rendering 综述（ar5iv 2302.00595）确认经典方法"放置由边缘检测/分割驱动，
优化方法评估标量 Lp 损失（典型 L2）"，**不**从逐像素误差图按方向选位置。
Image Painter（IEEE MultiMedia 2026, doc 11311375）的"CCL + PCA 笔初始化"与本流水线结构最像，
其 CCL 输入是**阈值化的标量幅值**（二值图），PCA 在每个连通分量上做——也是方案 A 路数。

### 2. 神经绘画：损失/评估用标量，残差作 3 通道仅喂给学习编码器

- **Paint Transformer (ICCV 2021, 2108.03798)**：`ℒ_pixel = ‖I_r − I_t‖_1`，标量 L1；无逐像素误差图、无空间注意力。
  笔颜色作为笔参数 (r,g,b) 直接预测，源自目标。
- **Compositional Neural Painter (MM 2023, 2309.03504)**：区域由网络回归矩形 (x,y,w,h)；奖励 `R = ‖I−C_t‖_2/‖I‖`，
  **标量 L2**（跨所有像素与 RGB 通道聚合），非逐通道/逐像素。笔颜色 (r,g,b) 直接预测。
- **Residual Guided Dynamic Queries (2025, researchsquare rs-9398429)**：直接相关。残差 `Id = It − Ic`（**3 通道**），
  但：① 训练/评估损失仍是标量 L1（`L_pixel = λp‖It − Ît‖_1`，Eq.4）；② 残差 3 通道喂给**学习编码器**得特征 Fd，
    再作 transformer 的动态 query——"在哪画"由**学习到的注意力**决定，注意力峰"对齐残差图的高误差区域"
    （即幅值语义），**不是**手工设定的 dRGB 聚类；③ 无 SLIC、无超像素、无手工聚类特征。

  → 神经路线把残差当 3 通道输入，但"选哪"用的是学习到的幅值注意力，损失用标量 L1。
    没有任何**手工**区域方法用 dRGB 作聚类键。

### 3. SLIC 本身：灰度是一等公民，距离公式与 m

SLIC 距离（Achanta et al., TPAMI 2012；多源复核）：

```
D = sqrt( dc^2 + (ds / S)^2 * m^2 )          # 等价于 sqrt((dc/m)^2 + (ds/S)^2)
S = sqrt(N / K)            # 网格步长，N 像素 K 超像素
m ∈ [1, 40]                # 紧凑度，越大越偏空间紧凑；自然图 Lab 常用 m≈10
```

- 论文原文强调 SLIC "seamlessly accommodates grayscale as well as color images"——灰度（1 值维 + 2 空间 = 3D）
  是原生支持模式，不是 hack。
- 灰度下 dc 是 1 维强度差；颜色下 dc 是 3 维 Lab 欧氏。
- 关键失衡：5 维时颜色占 3 维、位置 2 维，颜色项"票数"多，超像素更贴颜色边界、空间更碎，
  需调 m 或归一化。3 维（A）天然平衡。

### 4. 残差图 vs 自然图的语义不对称（用户假设的验证）

- **Vectorized Region Based Brush Strokes (arXiv 2506.09969)**：对**目标图**（自然图）做区域分割，
  用 **Lab 颜色空间**（6 处提及 Lab，1 处 RGB）——因为自然图的 RGB 是"外观"，按外观分组合理。
- 残差图则相反：值是"修正量"（标量 quantity）。语义差异**确实存在**，但方向**反着**：
  自然图→用颜色聚类；残差图→用幅值聚类。这恰好支持 A，而非 B。

## 定夺推理（综合）

1. **语义**：残差图每像素是"修正量"，量是标量。分组键 = "缺多少 + 在哪" = (x, y, v)。
   "缺什么色"的方向是**输出**（笔颜色），在区域定下后用区域内均值算即可，不必进聚类键。
   把输出量塞进聚类键是概念错位。

2. **方向与位置局部共线 → 方向是冗余信号**：迭代绘画中缺陷空间连贯；SLIC 紧凑度 m 约束下超像素空间紧，
   区内方向变化小 → 幅值聚类的区域已近似单方向，区内均值方向（笔色）有代表性。
   B 只在"区内方向真有变化"时才与 A 不同；而紧凑区内方向变化小，所以 B 大多时候要么与 A 同（常方向区），
   要么按噪声碎裂（低幅值区，反正不选）。对**被选中的高误差区**，B 相对 A 的净增益有限。

3. **权重平衡与跨迭代稳健性**：A 是 SLIC 灰度原生模式，2 空间 +1 值天然平衡，无重平衡。
   B 是 5 维，颜色 3 票 vs 位置 2 票，需重平衡；而残差值分布**每笔都变**（早期稀疏、晚期密集），
   一次性标定的权重会失效，要么每迭代重标定（成本/脆弱）要么用 SLICO 自适应归一化。
   下游有优化器精修，初始区域的**稳健性**比 B 的边际精度更重要。

4. **文献先例**：所有**手工**区域/放置方法用标量幅值（Haeberli L2、Image Painter 阈值+CCL、
   Paint Transformer L1）。3 通道残差只出现在**学习**编码器输入里（Residual Guided DQ），
   那里网络自行决定，不对应手工 SLIC 的特征选择。本流水线是手工 SLIC + 矩，结构最像 Image Painter → 走 A。

5. **颜色边界失效可检测、可下游救**：A 唯一真失效 = 被选区域跨目标颜色边界（两方向 → 脏色笔）。
   但 Image Painter 同样用标量阈值+CCL 仍达 SOTA，说明下游优化能修脏色；且该失效可由
   "选中区域内方向方差超阈"在线检测，触发时再按方向**只对该区域**二次细分即可，不必每迭代跑 5D SLIC。

## 逃生舱：切到 B 时的权重重平衡（公式级）

触发条件（任一）：① 选中区域内 dRGB 方向角方差 > 阈 τ（说明跨了颜色边界）；
② 实测笔颜色被均值成脏色且下游优化未能恢复。

重平衡（目标：把 3 维颜色距离拉回 1 维等效尺度，并按迭代自适应）：

```
D = sqrt( dc_norm^2 + (ds / S)^2 * m^2 )
dc      = ||dRGB||                         # 3 维残差欧氏距离
dc_norm = dc / (σ_c * sqrt(3))             # sqrt(3) 把 3D 距离拉到 1D 等效尺度
σ_c     = std( ||dRGB||  over 整个 D )      # 本迭代自适应，重算 → 抗稀疏→密集漂移
```

等价单 m 形式：

```
m' = m * sqrt(3) / σ_c
D  = sqrt( dc^2 + (ds/S)^2 * m'^2 )
```

或用 **SLICO 式每簇自适应**：dc 除以各簇局部颜色方差（去全局 σ_c 依赖，更稳）。

**残差图 m 取值建议**：比自然图 Lab 的 m≈10 **偏高**，取 m ≈ 20–40。
残差比外观更噪、空间紧凑对笔连贯更重要；调参时监控区域碎裂程度——要区域足够 elongated 容下一笔，
不要散成麻点。若仍碎，先升 m，再考虑 SLICO 自适应。

## 一句话交付

**A。** 残差是量非外观，按幅值+位置聚类；方向是输出（笔色）事后在区域内均值取；A 是 SLIC 灰度原生平衡模式、
跨迭代稳健、且与下游优化器配合（初始稳健 > 边际精度）。仅当跨颜色边界脏色且下游救不回时切 B，
按 `dc_norm = ||dRGB‖/(σ_c·√3)` 重平衡、m 取 20–40。
